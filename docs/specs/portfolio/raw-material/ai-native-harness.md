## 0. ★ 척추·배치 결정 (글 쓰기 전 반드시 반영)

> **★ 최종 발행 반영(2026-07-02) — 컨셉 B로 확정.** 발행본([ai-native-harness.mdx](../../../content/projects/ai-native-harness.mdx)) 제목 = **"구현자에서 설계자로 — AI 개발 하네스"**. 척추는 아래 A-프레이밍(하네스=메커니즘)이 아니라 **역할 전환 서사**(구현자→설계·오케스트레이터, 불가능한 마감이 강제한 전환점)로 갔고 **하네스는 그 전환의 how**로 재배치. 방어 가드: 레버리지-업(코딩 초월❌)·전환은 마감이 강제(트렌드❌)·설계/검증은 본인 몫·절대표현 금지. /codex 게이트 2회 통과(A 초안 게이트 + B 재구성 게이트). 아래 §0 A-프레이밍·§5 헤드라인 후보는 **초기안(참고용)**.

- **척추(thesis) [초기안 A]:** *"AI 산출물을 프로덕션에서 신뢰 가능하게 만든 하네스가 메커니즘이고, 물리적으로 불가능한 마감을 코드로 검증되는 규모의 인도로 바꾼 것이 결과다."* 속도는 후크가 아니라 결과. 후크는 **명명된 하네스**(case-study §2 P3). → *발행본은 이 위 B 노트대로 전환 서사로 재프레이밍됨.*
- **배치 [확정]:** overview 티어 대표작. 기존 발행 2편이 아래 증거로 매달린다 — [immutable-form-tree.mdx](../../../content/projects/immutable-form-tree.mdx)(협업양상 #1 깊은이해)·[dnd-fractional-indexing.mdx](../../../content/projects/dnd-fractional-indexing.mdx)(순수 본인 설계 균형추). **딥다이브의 메커니즘을 재탕하지 않는다** — evidence node로 **각 1문단 max**(immer=깊은이해 증거 · DnD=비-AI 순수설계 baseline) 후 링크로 내려보낸다(재교육 금지).
- **이 글의 기원 = "진짜 대표작" 예약** [확정·IV]: [questionnaire-builder-interview.md](./questionnaire-builder-interview.md) 메타에 "진짜 대표작은 AI 가속 + 프로덕션 레벨 하네스 세팅 서사, 이번 인터뷰에서 안 다룸"으로 이미 공백 표기됨. 이 파일이 그 공백을 채운다.

## 2. §9.1 원자료 폼

- **문제(실패 체인)** [확정·IV]: 기획자가 **PRD 문서만 계속 고도화** → 정작 Figma 화면은 빈약(기능 플로우·에러 처리 상세 화면 없음) → PRD는 도메인 용어 떡칠·가독성 낮음·구멍 많음·Figma와 불일치 → 기획자 마인드는 "PRD 다 줬으니 구현하면 되지" → **원 계획 개발기간이 워크타임 10일 미만**(주말·공휴일 제외, 규모상 물리적 불가). → 화면을 볼 개발자가 상세 플로우를 못 보는 상태로 불가능한 마감에 몰림.
- **재현 조건** [확정·IV]: 빈약한 Figma + 도메인 용어 PRD + 불가능 마감이 겹친 상황. 구멍난 화면을 중간에 딥하게 논의할 시간조차 없음(비효율).
- **영향 범위(정량 · canonical 실측 2026-07-02)** [확정]: 도메인 `src/features/custom` = **프로덕션 코드 ~29k LOC + 테스트 ~27k LOC = 도메인 전체 ~57k LOC**(테스트 밀도가 프로덕션과 맞먹음 = TDD 하네스의 증거 자체). 테스트 **167 파일**(74 `.test.ts` + 93 `.test.tsx`, `.spec` 0) · **~1,159 케이스** · 13 SSOT 규칙 · 9 슬라이스 · `temporal(persist(immer))`. 스토어 엔티티 슬라이스는 테스트:코드 159%. **글에서 "57k LOC"는 반드시 *테스트 포함 도메인 전체* 규모로 정직 표기**(프로덕션 코드만은 ~29k). ⚠️ 기존 [questionnaire-builder.md](./questionnaire-builder.md)의 "196 도메인 테스트[확정]"은 **실측(167)과 불일치 = 오류 → 167로 정정 대상**(그 파일도 고쳐 일관성 확보).
- **버린 대안 + 버린 이유** [확정·IV] (거절한 *운영모델* 3개 — 라벨 아니라 비용·실패모드로 기술):
  - ① *운영모델 A: PRD·Figma 구멍을 모두 회의로 선(先)해결 후 구현* → **버림**: 논의 지연이 마감을 통째로 먹고, 그래도 미문서 엣지는 남는다. 대신 **홀 케이스를 기능 플로우에 녹여 구현 → 개발 후 후속 정합**(속도 확보, 정합은 백로그로 이연 = 감수한 부채).
  - ② *운영모델 B: 개발자가 PRD를 수동 해석해 구현* → **버림**: PRD·Figma 드리프트로 **인간 해석 자체가 병목**이 된다. 대신 **스펙을 프로젝트에 내장해 드리프트 감지를 작업 루프 안으로**(H1 싱크 스킬의 기원).
  - ③ *운영모델 C: AI를 autocomplete 수준 보조로 제한*(문진빌더 이전 본인 방식) → **버림**: 처리량 목표상 AI가 **구현 초안을 대량 생성**해야 했고, 테스트·스펙이 그 **폭발반경을 제약**. 불가능 마감이 이 전환을 강제.
- **채택안 + 채택 이유** [확정·IV]: §1 하네스 7요소. 채택 근거는 취향이 아니라 **제약(불가능 마감)·리스크(AI 사이드이펙트 회귀)·병목(기획·API 소통)** 기준. TDD=회귀 리스크 흡수, 스펙 내장=기획 병목 흡수, 크로스레포=API 소통 병목 흡수.
- **검증 방식** [확정·IV]+[확정]: 테스트(=스펙)가 1차 방화벽 → 동등성 harness(신·구 어댑터 바이트 동일 roundtrip) → 본인 기능별 육안(도메인 제작자) → QA팀 게이트(본인+QA팀 공동). CI가 lint(error-only)+전체 테스트 그린 게이트.
- **배포 방식** [확정·IV]: H6 — 평행재구현 → 임시 진입점 QA 회귀 비교 → 정본 승격 → 레거시 삭제(가역). 초기 스프린트는 3주 풀타임(주말·공휴일 포함) 개발 → 4주차 개발·QA 병행.
- **결과(방어 가능)** [확정·IV]+[확정]: **canonical 앵커 문장(동일 범위 명시)** = *"문진 생성·편집·검증·저장·미리보기까지의 빌더를 원 계획은 영업일 10일 미만(주말·공휴일 제외)에 끝내는 것이었고 — 규모상 물리적으로 불가능 — 실제로는 3주 풀타임(주말·공휴일 포함) 개발 + 4주차 개발·QA 병행으로, **프로덕션 ~29k LOC + 테스트 ~27k LOC(167 파일·~1,159 케이스)·13 SSOT·9슬라이스** 규모를 인도했다."* ⚠️ "동일 범위"임을 글에서 못 박을 것(계획 범위 = 인도 범위). AI 귀속은 **일반 수준 유지**: "내장 스펙을 AI가 추적해 스펙 구멍을 제안, 테스트가 회귀를 막음"(특정 장면 귀속 금지 — §3). 하네스가 복리로 작동한 증거 = **V1에서 쌓인 테스트 커버리지를 지표 삼아 RHF→Zustand 재설계에 가속**([확정·IV], 상세는 questionnaire-builder-interview 토픽① 2차).
- **남은 비용·후속 리스크** [확정·IV]: ① **실제 발생한 정합 비용** — self-spec으로 메운 홀들이 부채로 남아 후속 정합 비용이 실제 발생(단 AI의 대가가 아니라 기획이 속도를 못 따라온 결과 — "빠른 속도가 만든 정합 백로그"로 정직 기술). ② **조직 재편 과제(진짜 리스크)** — 개발자가 기획·디자인까지 커버하고 속도로 압도 → 기존 워크플로우(기획→디자인→개발→QA) 고집 시 부적합. 포지셔닝 재정립이 남은 숙제 = 기술 부채 아닌 **조직 적응 비용**. ③ 코드로 확인된 커버 공백: **E2E 없음·성능 회귀 지표 없음·pre-commit 훅 없음**(에이전트 지적, [확정]). **핵심 재프레이밍**: 하네스가 *기술 리스크(회귀·드리프트)를 흡수*하고 남은 건 *조직 적응 비용*뿐 — "단점 없다"가 아니라 **"기술 대가를 조직 대가로 전환"**.
- **나 vs 팀 경계** [확정·IV]: 하네스의 **필요성·구성·운영 규율을 본인이 설계·도입**(감사 가능한 오너십 — "100%"는 이 지점에만 쓴다). 극한 개발밀도로 **팀 내 AI 최고 이해자로 부상** → 노하우 전파(하네스 장착·"사람이 병목 되지 마라"·설계자–AI 오케스트레이터 포지셔닝·plan-후-구현). 현재 강의로 베스트프랙티스 학습→즉시 적용→팀 상향평준화. QA만 본인+QA팀 공동.
- **공개 불가 정보(일반화할 것)**: 내부 티켓번호 · 커밋해시 · 병원/의료진 PII · 백엔드 클래스/경로 · 정확한 라우트명(→"임시 진입점") · 정확한 파일:라인. 의료 사전문진 도메인 민감.

## 3. 정직성 가드레일 (글에서 반드시 지킴)

- **AI 귀속은 일반 수준만** [확정·IV 합의]: "AI가 내장 스펙을 추적해 스펙 구멍을 제안 / 테스트가 회귀를 막음"까지만. **특정 홀·특정 회귀를 "AI가 바로 그걸 잡았다"고 붙이지 않는다**(기억 없음 = 지어내기 금지). 반증 가능성은 *AI 일화*가 아니라 **코드로 증명되는 하네스 산출물**(§5)로 채운다.
- **AI 의존의 정직한 경계** [확정·IV]: AI가 **구현 초안과 설계 후보를 대량 생성**, 본인은 **문제정의·후보 검증·정본 판단·회귀 방어선을 소유**. 그 깊이가 AI 가속의 상한. **"100% 의존"이라는 절대 표현은 감사 가능한 지점(하네스 도입 오너십) 외엔 쓰지 않는다.** (questionnaire-builder §1 N4 "설계 후보=AI·확정=본인"·immutable 글 "AI는 가속, 설계는 나"와 일관 — 모순 없이.)
- **"회귀 0" 표현 금지** [codex]: → **"알려진 회귀 없이 컷오버"** / "동등성 하니스·QA 비교로 주요 회귀 경로 차단". "0"은 측정 창·정의 없인 방어 불가. (발행 immutable 글의 "발견된 회귀 없이"와 일관 — questionnaire-builder의 "회귀 0" 문구는 이 글로 수입 금지.)
- **절대표현 금지** [확정·IV 합의]: "도태된다 / 단점 없다" 등은 §9.2 방어력 감점(오만으로 읽힘). **주장은 세게, 근거는 코드-검증 규모로** 받쳐 "확신에 근거가 있다"로. 본인도 동의("있는 그대로 전달력은 살리되 글 쓸 때 신경 쓰라").
- **속도 후크 주의** [확정·IV]: "한 달 만에 구현" 단독 후크 금지(그 버전을 재설계로 갈아엎음 → 면접 반격). 후크 = **불가능 마감 → 하네스 대응 → 코드-검증 규모 인도**.
- **codex 계열 절제(문진빌더 원자료 계승):** applied-engineering 톤, algorithmic research 아님.

## 4. 반증 가능 산출물 = "하네스의 이빨" (코드 확정 · AI 귀속 아님)

§3대로 AI 일화 대신 **하네스가 무엇을 지키는가**로 쓰는 코드-확정 재료. 회귀 가드(테스트가 **인접 변경에서 생기기 쉬운 회귀**를 막음 — AI 특정 귀속 아님) + 스펙-홀 메움(빈약 PRD/Figma 너머).

**회귀 가드(테스트 존재로 방어 — [확정]):**
- invalid rank sanitize: 백엔드가 깨진 rank 주면 "답변 추가" 시 키 생성 함수 throw → sanitize가 흡수(`sanitizeRanks` 테스트).
- 답변타입 파괴적 전환 데이터 손실: select(N옵션)→텍스트 전환 시 9개 메타필드 손실 → 동등성 roundtrip이 잡음.
- 저장 in-flight 가드: 저장버튼+이탈모달 저장 겹침 → 중복 PUT로 stale이 앞 변경 clobber → in-flight 가드(`useSaveDraft` 가드 테스트).
- 빈 카테고리 맨아래 drop 충돌 / 삭제된 카테고리 선택 고아 / legacy `type=null` 보존.

**스펙-홀 메움(빈약 PRD 너머 — [확정]):**
- **`gap-matrix.md` 문서 실재** = 홀들을 "정책 미정 / self-specced"로 추적 → Pillar 1("홀을 기능에 녹여 구현하고 후속 정합")의 실물 증거.
- FE가 BE보다 엄격: 드롭다운 **FE ≥2 vs BE ≥1**(빈 드롭다운 UI 방지) = PRD 미정의 규칙 자체 기획.
- 슬롯 모드 토글 백업 캐시(토글 시 데이터 손실 방지, PRD 미정의) · 자가복구 토스트(무음 자동복원의 "저장한 줄 착각" 약점을 3-gate 머지+1회 토스트로 가시화) · 빈 문항 저장 직전 자동 prune.
- SELECT+INPUT_TEXT `required=false` 규칙 = 백엔드 동작 역추론으로 FE 방어선 자체 기획.

### VDOC PRO PC — 의료진용 진료 데스크톱 앱 (2024.09 – 2026.04)

병원에 공급되는 의료진용 진료 보조 데스크톱 앱. 환자가 VDOC·PRO WEB으로 입력한 문진 데이터가 실시간 전달되어 진료 현장에서 즉시 활용됨. 환자 문진 → EMR 차팅까지 이어지는 파이프라인에서 의료진이 실제 진료를 보는 화면을 담당.

**주요 작업**

1. **Electron 자동 업데이트 — 클라우드/온프레미스 이중 체계**
   - 병원마다 방문 설치·업데이트하던 구조 개선. S3(클라우드)와 내부 서버(온프레미스)를 업데이트 피드로 분기하는 자동 업데이트 체계 구축.
   - 부팅 시 electron-updater 자동 버전 체크 후 다운로드·설치
   - 클라우드 병원 = electron-builder publish provider `s3`, 온프레미스 병원 = provider `generic`으로 내부 서버 URL 분기
   - Windows(NSIS)·macOS(DMG) 각각 빌드 타겟 대응
   - 결과: 클라우드는 S3 업로드 한 번, 온프레미스는 내부 서버 업로드 한 번으로 다수 PC 일괄 자동 업데이트

2. **실시간 환자 상태 소켓 알림과 네이티브 배지**
   - STOMP 소켓 이벤트를 IPC로 메인 프로세스에 넘겨 네이티브 Notification·배지로 전달하는 의료진 알림 시스템 구축.
   - STOMP 소켓으로 환자 상태 변경 이벤트 수신 (5초 간격 재연결, heartbeat, 최대 10회 재시도)
   - 전용 IPC 채널로 메인 프로세스 전달 → Electron 네이티브 Notification 표시
   - 알림 클릭 시 창 복원 + 해당 진료 탭 자동 포커싱
   - 단발성 알림 누락 방지용 앱 오버레이 배지 카운트

3. **치료가이드 템플릿 시스템**
   - CKEditor 기반 WYSIWYG 에디터 + 커스텀 비디오 플레이어로 의료진이 직접 치료가이드 작성·발송.
   - CKEditor 마크다운 툴바 에디터(노션 스타일, 실제 환자 노출 화면 그대로 편집)
   - 치료 영상 삽입 커스텀 비디오 플레이어(권장/비권장 상태 오버레이 배지)
   - 템플릿 CRUD·카테고리 폴더·검색 무한스크롤·미리보기·발송 흐름 통합

4. **개발 모드 — 서버 프로파일 동적 전환**
   - 빌드 분기 없이 단일 앱으로 클라우드·온프레미스 여러 병원 환경 QA.
   - 서버 프로파일 3종: publish(상용)·온프레미스 테스트·dev
   - 각 프로파일이 API URL + WebSocket URL을 한 세트로 묶음
   - 로그인 화면 드롭다운에서 즉시 전환, localStorage 저장

### VDOC PRO WEB — 환자용 문진 서비스 (2025.03 – 2026.03)

병원 내원 환자가 진료 전 AI와 대화하며 문진을 진행하는 병원별 맞춤형 웹. 환자 입력이 PRO PC로 실시간 전달되어 파이프라인의 시작점 담당.

**주요 작업**

1. **서버드리븐 UI 기반 확장형 문진 아키텍처**
   - 문진 구성·프로그레스·플레이스홀더·방문사유 흐름·커스텀 질문을 서버 스키마가 제어 → 병원·진료과별 커스텀을 프론트 배포 없이 런타임 반영.
   - UI 동작 11개 이상 + 질문 스키마 19여 개 필드를 서버가 제어
   - 1차·3차 병원 티어 분기도 서버 스키마 기반 런타임 반영
   - 기능 추가 시마다 프론트 배포가 따라가야 했던 구조적 한계 해결

2. **문진 시스템 V2 — 상태 기반 → 라우트 기반 재설계**
   - 단일 상태로 제어되던 V1을 라우트 단위로 쪼개 브라우저 히스토리와 동기화, 더티 체킹으로 AI 질문 컨텍스트 보존.
   - V1 한계: 모든 문진 페이지 단일 상태 → 브라우저 뒤로가기 시 전체 문진 유실, UI 백버튼도 현재 답변 유실
   - V2: 라우트 단위 페이지 분리 + 시퀀스 번호를 라우트 파라미터로 동적 렌더링 + 브라우저 히스토리 동기화
   - 문진 항목 타입 동적 렌더링: AI 텍스트 질문·통증 점수·커스텀 질문 12종
   - 더티 체킹 기반 답변 보존: 답변 미변경 시 이전 질문 컨텍스트를 Map에 저장해 재방문 복원
   - **설계 판단**: AI 문진은 이전 답변 컨텍스트로 새 질문을 생성. 답변이 같아도 AI는 매번 다른 질문을 던질 수 있어, 미변경 답변은 기존 질문까지 컨텍스트로 보장 → 뒤로 갔다 돌아오는 것만으로 흐름이 달라지는 현상 차단.

3. **소켓·폴링 병렬 실행으로 AI 문진 끊김 제거**
   - 다음 문진 조회에서 소켓의 끊김 취약성 + 폴링의 지연 딜레마를 병렬 실행 + 먼저 도착한 응답 채택으로 해결.
   - 폴링: 1초 주기, 최대 50회 자가복구
   - 소켓: 5초 간격 재연결, heartbeat, 최대 10회 재시도
   - 답변은 Map 컨텍스트에 저장해 폴링 머지 시 덮어쓰지 않도록 설계

4. **문진 중 의도치 않은 이탈 방지**
   - TanStack Router `useBlocker`로 문진 외부 이동 가로채 컨펌 모달로 의도 확인. 내부 이동은 자연스럽게 유지, 플로우 이탈 시에만 컨펌.

### VDOC (B2C 웹) — https://ai.vdoc.kr/ (2025.05 – 2025.11)

의료 특화 AI에게 증상 입력 시 원인 분석·건강 리포트를 받는 B2C 웹. 서비스 내 병원 접수로 문진 데이터가 PRO PC까지 이어짐. 초기부터 설계·구축해 핵심 사용자 여정 end-to-end 완성, B2C 론칭 프론트 기반 마련.

**주요 작업**

1. **비회원→회원 자동 마이그레이션 + 팝업 인증**
   - 게스트가 인증 지점에서 이탈하는 문제를 팝업 인증 + 데이터 자동 마이그레이션 + 화면 복귀로 해결.
   - `window.open` + `postMessage` 팝업 인증으로 현재 화면 유지한 채 비동기 로그인
   - 로그인 성공 시 게스트 → 회원 계정 자동 마이그레이션
   - 별도 리다이렉트 스토어로 원래 화면 무중단 복귀

2. **멀티 지도 프로바이더를 전략 패턴으로 추상화**
   - Naver·Google을 공통 `MapEntity` 인터페이스로 추상화 → 프로바이더 추가/유지보수가 동일 규격 위에서.
   - 배경: Naver=한국 친화, Google=해외 확장·거주자 병원 접수
   - 전략 패턴 `MapEntity`: 지도 생성·마커 생성·중심 이동·줌 변경 등 7개 공통 메서드로 Naver·Google 구현체 통일
   - 마커 렌더링 부하 제어: 줌·뷰포트 기준 표시 수 제한, 위치 변경 시 재생성+정리
   - 지도를 라우트 바탕 레이어로 두고 리스트·상세·접수를 Outlet 레이어로 → 페이지 이동 시 재렌더 방지
   - 지도 현재 위치를 쿼리 파라미터와 동기화 → 위치 복원 + URL 직접 위치 전달
   - **설계 판단**: 가장 경계한 것은 페이지 이동마다 지도 재렌더 비용. 라우트 바탕 고정 + Outlet + 쿼리 동기화 조합으로 위치 유지 + 병원 위치 공유 확장성 확보.

3. **다중 OAuth + 토큰 리프레시 큐 + 다중탭 세션 동기화**
   - OAuth 3종(카카오·애플·PASS)을 단일 `AuthProvider` 추상화로 통합
   - 동시 401 발생 시 `failedQueue` 기반 토큰 리프레시 큐로 중복 요청 차단
   - localStorage 이벤트 + Zustand persist로 다중탭 세션 동기화

4. **한국어 키 기반 i18n 구조 + 언어 우선도 설계**
   - 키 네이밍을 영어 코드가 아닌 한국어 원문으로 → 화면·코드 탐색 용이, 중복 문구 단일 키, 번역 부담 경감
   - 스캐너 스크립트로 코드베이스↔번역본 자동 동기화(추가/삭제)
   - 우선도: 로컬스토리지(URL lang 쿼리·사용자 변경 기록) > 브라우저 언어, 지원 밖이면 영어 폴백

환자가 증상을 표현하고 의료진이 진료 시 쓰는 인체도가 PRO PC·PRO WEB·VDOC 세 제품 모두에 필요. 여러 제품에 반복돼도 유연하게 붙는 구조로 설계 + 이미지 에셋 경량화로 빌드 용량 절감.

- 코어 로직 추상화: 세 제품이 동일 인체도 설계 공유, 각 제품 비즈니스 로직 접점만 독립 구현
- React Context 스코프 제한: 인체도 영역 내부에서만 유효(줌·부위 클릭·오버레이 등 제어) → 전역 사이드이펙트 방지
- 진료과별 분기 로직 + 내부 매핑: 진료과-부위 매핑, 모든 부위에 서버·AI 소통용 고유 ID 매핑 → 선택 부위가 문진·의료진 화면까지 이어짐
- **빌드 용량 절감: PRO PC 기준 21,410 kB → 2,959 kB (약 86% 감소)**, 다수 인체 부위 이미지 에셋 경량화
- **설계 판단**: 초기엔 PRO PC 하나였지만 인체도가 다른 제품에도 들어갈 가능성을 일찍 판단해 처음부터 추상화 투자. 미투자 시 제품별 상이 설계 → 유지보수 비용 기하급수 증가.

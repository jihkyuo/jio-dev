# 상세 페이지 스캐너 어필 강화 — 검증된 진단·설계

> 작성 2026-07-01 · 근거: 발행 글 2편 코드 추적 + codex(gpt) 교차검증 1회
>
> 전제: 케이스 스터디는 정독되지 않는다(NN/g 79% 스캔). 골격([case-study-structure.md](./case-study-structure.md) §4)은 이미 스캔-우선이다. 이 문서는 **작성 규범은 그대로 두고, 렌더링 층에서 새는 스캐너 어필**만 좁혀 고친다.

## 진단 (codex 교차검증 완료)

| # | 누수 | 성격 | 상태 |
|---|---|---|---|
| A | **모바일에서 5초 페이로드(요약표)가 목차 카드 밑으로 밀린다.** `page.tsx`가 `lead → <Toc/> → body(## 요약~)`로 주입 → 모바일 인라인 목차(`xl:hidden`)가 요약표 앞을 막음. 데스크톱은 목차가 `position:fixed`라 무관. | 구조·동선 파괴 | **최우선** |
| B | 큐레이션 한 줄 `impact`가 카드·호버프리뷰엔 뜨지만 상세엔 없음(architecture.md에 "strip 제거됨"으로 의도적 제거 기록). 착지한 스캐너가 토픽만 얻고 결과 verdict를 못 가져감. | 구조 | 높음 |
| C | 헤딩 `## 요약`·`## 딥다이브`가 제목만 읽는 경로(특히 목차)에서 정보량 0. 데스크톱 목차 아코디언이라 `딥다이브` 자식이 접혀 더 나쁨. | 구조·작음 | 높음 |

**교정(codex)**: 초기 가설의 "데스크톱 목차 잘림"은 오진 — `left-[min(…,100vw-14rem)]`로 클램프돼 안 잘림. 진짜 문제는 xl 미만 상주 목차 부재이며, 이는 폴리시로 분류해 이번 범위에서 제외.

## 설계

### Fix A — `<Toc/>` 주입점을 첫 h2(요약) 섹션 뒤로

- [splitLeadAndBody.ts](../../../src/shared/lib/splitLeadAndBody.ts)를 3분할로 일반화: `head`(첫 h2 전) · `summary`(첫 h2 ~ 둘째 h2 전) · `rest`(둘째 h2~).
- [page.tsx](../../../src/app/projects/[slug]/page.tsx): `source = head + summary + "<Toc/>" + rest`.
- 모바일 동선: `H1 → 칩·도입 → 요약표 → 목차 → 핵심원인…`. 데스크톱 영향 없음(fixed 사이드바).
- **경계 근거**: `## 요약`은 스펙상 항상 첫 섹션이라 견고. 본문 제목은 글마다 달라 3번째 h2 기준은 취약 → "둘째 h2 앞"이 안전점.
- **엣지**: 첫 h2 없음 → 현 동작(목차 없음) 유지. 단일 h2 → `rest` 빈 문자열, 목차는 끝에.
- **검증**: 3분할 순수 로직 RED-first Vitest(단일 h2·h2 없음·펜스 안 `##`). 렌더 순서는 모바일 수동 검증(컴포넌트 하니스 없음).

### Fix B — `impact`를 H1 아래 deck(부제)로 재도입, 중복 회피

- `meta.impact`를 chrome(page.tsx) H1 바로 아래 한 줄로 렌더. 스타일은 카드와 통일(`font-mono text-accent` 계열).
- 순서: `H1 → impact deck → [MDX: 칩 → 도입단락]`.
- **중복 회피(핵심)**: deck = 0초 verdict(압축 결론), PAAR R행 = 5초 방어가능 디테일(Before→After·제거한 위험). 같은 결과를 **해상도만 다르게** 두는 역피라미드라 §4 "같은 문장 3회 재탕" 금지에 해당 안 됨. 글 본문 수정 불필요(R행이 이미 impact보다 상세).
- **판단 포인트**: 실제 렌더에서 deck·R행이 "다른 해상도"로 읽히는지 확인. 재탕처럼 느껴지면 deck 톤을 "왜 중요한가"로 틀거나 접는다.
- architecture.md의 "strip 제거됨" 문구를 갱신(의도된 재도입 기록).

### Fix C — 죽은 라벨 제거

- `## 요약` → 최소 변경(표 자체가 요약). 필요시 `## 3줄 요약`으로 스캔 신호 부여.
- `## 딥다이브` → 글이 무엇을 증명하는지 담은 라벨로. 예: immutable `## 설계를 코드로 — 다섯 기능이 한 원리 위에`, dnd `## 구현 디테일 — rank가 새지 않게`. (문구는 구현 시 확정.)
- 구조 라벨 금지 규칙을 [case-study-structure.md](./case-study-structure.md) §4에 추가 → 다음 글 자동 준수.
- **검증**: MDX 편집 후 `pnpm content:check` + 목차 렌더 수동 확인.

# 콘텐츠 이식성·거버넌스 (B안) — Spec

> 작성일 2026-06-30 · 작성자 핸드오프(다른 세션이 실행)

## 무엇을 (What)
포트폴리오 케이스 스터디 문서를 (1) "콘텐츠가 조용히 코드가 되는" 드리프트를 **포맷·CI로 차단**하고, (2) 소스 포맷을 **MDX → Markdown + remark-directive**로 이행해 **툴 간 이식성**과 **미래 WYSIWYG 편집기 친화성**을 확보한다.

## 왜 (Why)
이 사이트는 개인 자산으로 수십~수백 개 포트폴리오를 담을 예정. 소유자 우선순위는 (a) 콘텐츠-not-코드 유지보수성, (b) 노션/컨플루언스 등과의 문서 이동 이식성, (c) 추후 WYSIWYG 편집기 추가 가능성 — 렌더 유연성보다 위다.

현재 진단(두 모델 합의):
- 문서는 **지금은 content-driven**(라우트 1개 `[slug]`, 새 글 = `.mdx` 1장, 코드 0줄). page-per-item 아님.
- 그러나 **MDX는 임의 JSX/import/표현식을 허용** → 포맷이 "콘텐츠만" 규율을 강제하지 않음. 규율이 사람 판단에만 의존 = 드리프트 리스크.
- MDX는 **인바운드 붙여넣기에 취약**(생 HTML·stray `<`/`{`에 빌드째 깨짐)하고 **아웃바운드 라운드트립 불가**(커스텀 JSX는 다른 툴에서 글자로 깨짐).
- MDX-소스는 **WYSIWYG 라운드트립을 어렵게** 함(임의 코드→편집 모델 역변환 불가).

상세 근거: 이 대화의 아키텍처 분석 + codex consult 감사 결과(아래 plan.md "근거" 참조).

## 요구사항 (Requirements)
- **R1 (가드레일):** 콘텐츠 파일 본문에 `import`/`export`/JSX 표현식/`style=`/`className=`/`<script>`/허용목록 밖 대문자 컴포넌트/위험 raw HTML가 들어오면 **CI·로컬 체크에서 실패**한다.
- **R2 (사전 컴파일 검증):** 잘못된 본문(stray `<`/`{`, 잘못된 컴포넌트 prop)이 **배포 빌드가 아니라 `content:check` 단계에서** 먼저 걸린다.
- **R3 (소스 포맷 이행):** 커스텀 블록을 JSX가 아니라 **디렉티브**로 저작한다 — `:::callout{type="decision"}` / `::chip[…]` / `:::meta` / 인라인 `:hl[…]`.
- **R4 (인바운드 견고성):** 외부 표준 마크다운(생 HTML·`<`/`{` 포함)을 붙여도 빌드가 안 깨지고 우리 G2 스타일로 렌더된다(rehype-raw + rehype-sanitize).
- **R5 (시각 동일성):** 이행 후에도 **현재 G2 Tintglass 렌더 결과가 동일**해야 한다(콜아웃·표·인용·칩·코드·헤딩·제목).
- **R6 (기존 글 이행):** `content/projects/*.mdx` 3개를 `.md` + 디렉티브로 변환.
- **R7 (단어 하이라이트 — 디자인 확정):** 본문 특정 단어를 강조하는 하이라이트를 지원한다. **시안 확정 = A Marker**(accent 풀하이트 틴트 마커, 보더 없음, 흰 글자). 저작은 인라인 디렉티브 `:hl[단어]` / `:hl[단어]{c="decision"}`(시맨틱 색). 더불어 **인라인코드를 코랄-오렌지로 분리**(`--code: #e0966b`)해 하이라이트(블루)와 색 역할을 구분한다. 근거·대안 비교: 이 spec과 같은 디렉토리 plan.md "하이라이트(R7)" 절 + 프로토타입 `scratchpad/design-variants/highlight.html`.

## 성공 기준 (Acceptance)
- `pnpm content:check`가 위반 콘텐츠(예: `import`, `<script>`, 미허용 `<Foo>`, `style=`)에서 **실패**한다(테스트로 증명).
- 잘못된 본문이 `content:check`에서 잡히고, 정상 빌드(`pnpm build`)는 통과.
- 외부 마크다운 샘플(생 HTML·`<`/`{` 포함)을 콘텐츠로 넣어도 빌드 통과 + 표준 요소가 G2 톤으로 렌더.
- 디렉티브로 작성한 콜아웃/칩/메타/하이라이트가 기존 JSX 버전과 **시각적으로 동일**.
- 기존 3개 글이 `.md`로 이행되어 동일하게 렌더 + 기존 테스트 전부 그린 + 빌드 그린.
- **TOC(목차)·스크롤스파이가 회귀 없이 동작**(헤딩 id/스크롤스파이 동일).
- (R7) `:hl[…]`로 쓴 하이라이트가 A Marker 스타일(블루 틴트 마커)로 렌더되고, `{c="decision"}` 등 시맨틱 색이 적용된다. 인라인코드는 코랄(`#e0966b`)로 보이며 하이라이트와 색이 구분된다.

## 범위 밖 (Non-goals)
- WYSIWYG 편집기 **구현**(이 작업은 "편집기가 쉬워지는 토대"까지만; 편집기 자체는 별도).
- G2 시각 디자인 **변경**(톤 유지가 목표지 재디자인 아님).
- **TOC 위젯 수정 금지** — `src/widgets/toc/`, `src/shared/lib/extractHeadings.ts`, `src/shared/ui/HeadingAnchor.tsx`는 불가침. 헤딩 `id`(github-slugger via rehype-slug)·`scroll-mt-24`·`group` 보존.
- 아웃바운드 픽셀 유지는 비목표(디렉티브의 텍스트 격하로 충분 — 소유자 합의).
